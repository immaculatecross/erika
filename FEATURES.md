# Milestones

In order; each assumes the previous. Statuses: `backlog` → `next` → `building` → `done`. **v0.1 is E-1 through E-5** (D-5, D-12 — the operator's call). One milestone = one mission handed to the dispatcher.

| ID | Status | Milestone | Acceptance test |
|---|---|---|---|
| E-1 | done | Foundation — shell & design system | `npm run dev` serves the app; sidebar nav (Sessions, Practice, Settings) with empty states obeying DESIGN.md; page transitions and list-stagger animations present (Motion), `prefers-reduced-motion` respected; SQLite via a migrations runner; Settings persists target language, native language, model tier, and monthly budget across reloads; `lint`/`typecheck`/`test`/`build` green locally and as CI checks on PRs; a Playwright screenshot script captures any route to a file. |
| E-2 | building | Capture — sessions in | Record from the mic with live level meter and elapsed timer (chunked MediaRecorder — long takes survive); upload mp3/wav/m4a/webm/ogg/aac/flac up to 24 h / 2 GB, streamed to disk (never buffered in memory); duration probed via ffprobe, over-cap files rejected with a truthful message; sessions list + detail with player; ingest-job rows with visible progress states; delete removes rows and files. |
| E-3 | backlog | Smart ingest — only the speech | Every capture runs an async job: normalize to 16 kHz mono; voice-activity detection extracts speech segments with original timestamps and discards silence/noise; segments under 2 s dropped; each segment content-hashed for dedup and caching; session page shows raw vs speech time ("6 h dump → 47 min speech") and a segment timeline; time-compressed triage renditions (configurable 1.25–1.5×) produced per segment; a day-long fixture processes without exhausting memory and a killed job resumes where it stopped. |
| E-4 | backlog | Analysis — the cascade coach | Analyze runs the cascade: `gpt-audio-mini` triages compressed segments and flags suspicious windows; `gpt-audio-1.5` deep-listens only the flagged windows at native speed; findings persisted (quote, correction, category grammar/vocabulary/phrasing/idiom/pronunciation, explanation, severity, timestamp) focused on the dominant speaker; a cost estimate from a rates table is shown before the run and the monthly budget cap halts runs truthfully; progress visible without reload; report UI with per-category counts, expandable findings, jump-to-audio at the finding's timestamp; cached segments are never re-billed; parsing covered by fixture tests plus one real-API smoke run documented in the PR. |
| E-5 | backlog | Flashcards — drill your own mistakes | Every finding generates a card (front: your phrase in context; back: correction + why), deduplicated; Practice shows the due queue; full-screen practice with 3D flip and Again/Hard/Good/Easy grading persisted through SM-2 scheduling; keyboard shortcuts (space to flip, 1–4 to grade); card browser with suspend and delete; export to CSV importable by Anki. **End of v0.1.** |
| E-6 | backlog | Micro-lessons | Recurring error patterns become short grammar lessons with interactive exercises (multiple choice, fill-in, rewrite graded by the model with feedback); completing a lesson updates pattern mastery. |
| E-7 | backlog | Focus map & progress metrics | Error rate per speaking-hour by category, trended across sessions; ranked "what to work on next"; the whole answer on one screen. |
| E-8 | backlog | Pronunciation studio | Re-record a corrected phrase; the audio model compares your take against the target and gives specific feedback; minimal-pair drills generated from your pronunciation findings. |
| E-9 | backlog | Phrasebook — the recast library | A searchable bank of "you say X, natives say Y" built from findings, including full-utterance recasts side by side; pin any entry into flashcards. |
| E-10 | backlog | Conversation gym | Realtime voice roleplay (`gpt-realtime`) that steers scenarios into your weak patterns; a gym session yields findings like any capture. |
| E-11 | backlog | Speech archive | A searchable timeline of your analyzed speech (transcripts exist only as analysis byproducts), filterable by category and severity — your speaking life at a glance. |
| E-12 | backlog | The editor's letter | A weekly digest: trends, best recasts, the one thing to focus on next week. Quiet ritual, no gamification. |
| E-13 | backlog | Voice enrollment — only you | A 30-second enrollment teaches Erika your voice; day dumps are filtered to your speech before analysis — bystanders are never analyzed (privacy) and never billed (cost). |
| E-14 | backlog | Hosted + OAuth | The same app deployed multi-user behind OAuth sign-in. |
| E-15 | backlog | Companion & mobile | An authenticated ingestion API an always-on recorder pushes to; an Expo app sharing the same routes — Erika without a browser. |
