# Milestones

Statuses: `backlog` → `next` → `building` → `done`. One milestone = one mission handed to the dispatcher. Version scope: **v0.1 = E-1…E-5** (D-5, D-12) · **v0.2 = E-7, E-9, E-11, E-6, E-12** in that build order (D-15) · **v0.3 = E-16…E-23** in ID order (D-16, ratified from RETRO-001). E-1…E-15 are listed in original roadmap order; the build order is whatever the version decision names.

| ID | Status | Milestone | Acceptance test |
|---|---|---|---|
| E-1 | done | Foundation — shell & design system | `npm run dev` serves the app; sidebar nav (Sessions, Practice, Settings) with empty states obeying DESIGN.md; page transitions and list-stagger animations present (Motion), `prefers-reduced-motion` respected; SQLite via a migrations runner; Settings persists target language, native language, model tier, and monthly budget across reloads; `lint`/`typecheck`/`test`/`build` green locally and as CI checks on PRs; a Playwright screenshot script captures any route to a file. |
| E-2 | done | Capture — sessions in | Record from the mic with live level meter and elapsed timer (chunked MediaRecorder — long takes survive); upload mp3/wav/m4a/webm/ogg/aac/flac up to 24 h / 2 GB, streamed to disk (never buffered in memory); duration probed via ffprobe, over-cap files rejected with a truthful message; sessions list + detail with player; ingest-job rows with visible progress states; delete removes rows and files. |
| E-3 | done | Smart ingest — only the speech | Every capture runs an async job: normalize to 16 kHz mono; voice-activity detection extracts speech segments with original timestamps and discards silence/noise; segments under 2 s dropped; each segment content-hashed for dedup and caching; session page shows raw vs speech time ("6 h dump → 47 min speech") and a segment timeline; time-compressed triage renditions (configurable 1.25–1.5×) produced per segment; a day-long fixture processes without exhausting memory and a killed job resumes where it stopped. |
| E-4 | done | Analysis — the cascade coach | Analyze runs the cascade: `gpt-audio-mini` triages compressed segments and flags suspicious windows; `gpt-audio-1.5` deep-listens only the flagged windows at native speed; findings persisted (quote, correction, category grammar/vocabulary/phrasing/idiom/pronunciation, explanation, severity, timestamp) focused on the dominant speaker; a cost estimate from a rates table is shown before the run and the monthly budget cap halts runs truthfully; progress visible without reload; report UI with per-category counts, expandable findings, jump-to-audio at the finding's timestamp; cached segments are never re-billed; parsing covered by fixture tests plus one real-API smoke run documented in the PR. |
| E-5 | done | Flashcards — drill your own mistakes | Every finding generates a card (front: your phrase in context; back: correction + why), deduplicated; Practice shows the due queue; full-screen practice with 3D flip and Again/Hard/Good/Easy grading persisted through SM-2 scheduling; keyboard shortcuts (space to flip, 1–4 to grade); card browser with suspend and delete; export to CSV importable by Anki. **End of v0.1.** |
| E-6 | done | Micro-lessons | Recurring error patterns become short grammar lessons with interactive exercises (multiple choice, fill-in, rewrite graded by the model with feedback); completing a lesson updates pattern mastery. |
| E-7 | done | Focus map & progress metrics | Error rate per speaking-hour by category, trended across sessions; ranked "what to work on next"; the whole answer on one screen. |
| E-8 | backlog | Pronunciation studio | Re-record a corrected phrase; the audio model compares your take against the target and gives specific feedback; minimal-pair drills generated from your pronunciation findings. |
| E-9 | done | Phrasebook — the recast library | A searchable bank of "you say X, natives say Y" built from findings, including full-utterance recasts side by side; pin any entry into flashcards. |
| E-10 | backlog | Conversation gym | Realtime voice roleplay (`gpt-realtime`) that steers scenarios into your weak patterns; a gym session yields findings like any capture. |
| E-11 | done | Speech archive | A searchable timeline of your analyzed speech (transcripts exist only as analysis byproducts), filterable by category and severity — your speaking life at a glance. |
| E-12 | done | The editor's letter | A weekly digest: trends, best recasts, the one thing to focus on next week. Quiet ritual, no gamification. |
| E-13 | backlog | Voice enrollment — only you | A 30-second enrollment teaches Erika your voice; day dumps are filtered to your speech before analysis — bystanders are never analyzed (privacy) and never billed (cost). |
| E-14 | backlog | Hosted + OAuth | The same app deployed multi-user behind OAuth sign-in. |
| E-15 | backlog | Companion & mobile | An authenticated ingestion API an always-on recorder pushes to; an Expo app sharing the same routes — Erika without a browser. |
| E-16 | done | Hardening & integrity | Cache-reused findings carry timestamps remapped onto the target session's segment (regression test with an offset-shifted duplicate hash); job claim uses a heartbeat lease so a second worker cannot re-run a live job, plus an insert-if-absent guard on `findings`; VAD splits any speech interval over `MAX_SEGMENT_MS` at its quietest contained dip so no segment is buffered whole; spend is recorded when a billable call *resolves*, not after parse (a parse failure still ledgers the charge). **Part 2 — the operator's own session:** `npm run worker` loads `.env.local` and exits non-zero without an API key; a job sitting queued with no live worker heartbeat says so in the UI; VAD thresholds on the recording's *measured* noise floor with pre/post-roll padding and a widened merge gap, calibrated against a committed labelled sample (recall asserted, D-13); a truncated or unparseable model reply gets one bounded repair retry and then marks only *that* segment unreadable while the run completes and reports the count; Analyze is refused (409) and not offered on a session with no segments; a lost mic take says so; the polling hooks stop on 404/410; the pre-commit hook blocks raw NUL bytes in source. |
| E-17 | done | One findings truth | A single `lib/findings-model.ts` defines "analyzed session" and "included finding"; all six read sites consume it and agree on the same week's counts (test covering the previously-divergent halted-run and re-analysis cases); focus/letter aggregates computed in SQL rather than per-session loops with identical results; a schema/ER doc lands and is bound to the migration ritual. |
| E-18 | done | The honest home | `/practice` composes a daily plan (due cards + the lesson Focus's ranking prescribes + the unread letter) instead of an interstitial; sessions rows show yield (speech time, findings, dominant category) and unanalyzed sessions are visually distinct with an inline analyze affordance showing its estimate; Settings shows month-to-date spend against the cap; lesson rows state "Lesson ready" vs "Generate — est. $X"; green is reserved for resolved/mastered/improving (LOW and READY badges lose it). |
| E-19 | done | Profile-primed analysis | A compact speaker profile (native language, top recurring quote→correction pairs, category rates, mastery) is built without a model call and injected into triage, deep-listen, and lesson prompts (fixture tests assert the L1 and profile lines); the deep model marks findings that recur a profile entry and the recurrence link is persisted. |
| E-20 | done | Slips — the fossil dossier | Findings cluster into persistent slips with a stable key; each slip's dossier interleaves every occurrence (jump-to-audio) with its drill history (card grades, lesson completions) on one timeline; a slip's state computes as active fossil / in remission / resolved from later analyzed sessions containing no matching occurrence ("Not heard since 13 Jul · 3 sessions clean"); Focus shows a resolved count and green finally means mastery. |
| E-21 | done | Contrastive playback | Each finding's correction is rendered once in the audio model's voice, cached per finding and recorded in the shared spend ledger; a Compare control on card backs and phrasebook rows plays your clip then the native rendering; replays make zero model calls (exactly one ledger row per finding); the budget cap refuses generation truthfully. |
| E-22 | done | The session map | The session timeline plots severity-tinted finding markers on their segments, clicking a segment plays it and selecting a finding highlights its segment; Focus gains a "when you slip" distribution by wall-clock hour from a unit-tested pure function. |
| E-23 | done | Ask Erika | An "ask for more" affordance on any finding returns a persisted deeper note citing at least one other finding from the user's own corpus; re-opening it makes zero model calls; budget-capped and ledgered. |
