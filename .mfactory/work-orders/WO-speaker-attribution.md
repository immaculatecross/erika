## Objective
On-device **speaker attribution**: using the enrolled user's voice (the E-35 enrollment take), decide **per segment** whether the speech is the enrolled user, so that **produced-lemma positive evidence is only minted for the user's own speech** — a bystander/podcast/other person is never credited as the user, and (D-22 **recall-first**) the user's own speech is **never dropped**. All acoustic processing is **on-device**; enrollment and session audio never leave the machine. Also close the two RETRO-002 owed items that live on this exact path: a **produced-lemma idempotency key** (replay-safe emission) and a **session-level exclusion surface** ("this recording isn't me — don't learn from it").

## Acceptance criteria
1. **A `SpeakerEmbedder` seam, real impl live-gated, mock-tested (mirror `AudioModelClient`).** Define an interface (e.g. `embed(pcmWindow): Float32Array` + a `compare`/cosine similarity) threaded as a **parameter** into the attribution stage — never imported inside the pipeline. The **production implementation** is a local **sherpa-onnx** speaker-embedding model (CAM++/WeSpeaker class, ~29 MB `.onnx`) loaded via the **`#47`-safe module-relative asset discipline** and traced into the standalone build; add the dependency. Because the sandbox has **no egress and cannot download/run the model**, the live sherpa path is **operator-gated** (guard behind availability; degrade honestly if the model asset is absent — never crash ingest). Tests use a **mock embedder**.
2. **On-device attribution stage — windowed, recall-first.** Insert an `attributing` stage after `segmenting` in the ingest pipeline (resumable/checkpointed like the others). For each segment, score **short sliding windows (~3–5 s)** against the **enrolled user reference embedding** (compute + cache the reference from `latestEnrollment`; recompute on re-enrollment). Persist a per-segment verdict (**migration v23**: e.g. `segments.speaker_score REAL`, `segments.is_user INTEGER` — nullable; **null = unattributed**). **Recall-first (D-22):** the threshold τ is calibrated so **user-recall ≥ 0.99** (never exclude the user), explicitly accepting more false-includes; document this tradeoff in code. **Graceful degrade:** if there is **no enrollment**, or the model asset is unavailable, attribution is skipped and every segment is treated as the user (verdict null / is-user) — attribution is a best-effort *filter*, never a gate that can silence a learner who hasn't enrolled.
3. **D-13-honest calibration against a REAL two-voice fixture.** Commit a small, **license-clean two-real-speaker** fixture with per-window speaker labels (mirror `make-labelled-speech.sh`/`labelled-speech`). Calibrate τ on it and **falsify a naive baseline** (e.g. a midpoint τ that drops some user windows → user-recall < 0.99, or a "mark everything user" straw man that never excludes anyone): a test asserts the calibrated τ reaches ≥0.99 user-recall **while actually excluding** the other speaker's windows at a stated rate, and that the naive baseline fails one of those. **Strongly prefer** an embedding that **runs in the sandbox** (e.g. a deterministic ffmpeg-derived acoustic feature) so the calibration is genuinely exercised end-to-end, with sherpa-onnx as the quality upgrade behind the same interface. If a real in-sandbox embedding is truly infeasible, a **documented synthetic-embedding** calibration is acceptable ONLY if it still falsifies a naive baseline with realistic within/between-speaker overlap (not trivially separable) — and you must state plainly that **live τ re-calibration against the real model is operator-gated** (D-19 honesty; do not oversell). State which path you took and why.
