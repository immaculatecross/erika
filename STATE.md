- **E-17 DONE — one findings truth.** Six surfaces answered "what are the user's findings?" six ways and disagreed: Focus and the letter gated on the *latest* analysis job being `done`, so a budget-**halted** run or an enqueued **re-analysis** deleted a whole session from them while the Phrasebook, Archive, patterns and cards kept showing it — the letter could report 3 findings for a week the Phrasebook showed 9 of. **`lib/findings-model.ts` now defines the scope once** and all six read sites consume it. The semantics, chosen for truthfulness: the atom is an **analysed segment** (its `content_hash` carries a complete `segment_analyses` witness — triaged, deep-listened if flagged, not unreadable); an **included finding** is one whose own audio is analysed; an **analysed session** has ≥1 analysed segment *and* ≥1 analysis run of its own past `queued` (witnesses are hash-shared across sessions, so a byte-identical re-upload contributes nothing anywhere until its own Analyze runs — a pure cache hit that then counts instantly) and is denominated on *only* its analysed speech. **Job state never un-says committed evidence** — a halted run counts (over the hours actually heard, never inflating the rate) and a re-analysis in flight changes nothing while it runs. Focus and the letter collect in **two SQL `GROUP BY` queries each** instead of three per session, proven identical to the old collectors on cleanly-`done` fixtures. The session report's tally is counted from witnesses, not `segmentCount − unreadableCount`, which claimed "5 of 6 analysed" on a halted run that heard 1. **`docs/schema.md`** documents v1–v9 and `tests/migrations.test.ts` fails if a migration lands without it.
