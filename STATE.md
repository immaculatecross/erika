**E-2 Capture shipped.** Both ways in are live. Upload backbone (part 1): `POST /api/sessions` streams an audio file to disk under `data/sessions/<id>/` (never buffered — a byte cap aborts mid-stream), `ffprobe` probes duration (D-7; over-24 h/2 GB rejected and cleaned up), and it becomes a session row with a `queued` ingest job. Mic capture (part 2): a Record control beside Upload records live from the mic — a chunked `MediaRecorder` (1 s timeslices, so long takes survive), an elapsed timer in tabular numerals, and the signature level meter that breathes with the voice off a Web Audio `AnalyserNode` (spring on `scaleY`, degrading to a non-animated height readout under `prefers-reduced-motion`). On Stop the ordered chunks assemble, are decoded and re-encoded to WAV client-side (a live WebM carries no container duration ffprobe can read), and POST to the same endpoint — landing in the list with a `queued` job. Denied/unsupported mic renders a quiet, specific line; Upload keeps working. Pure logic (elapsed format, level-from-analyser, chunk assembly, mime pick, WAV encode) lives in `lib/recording.ts` (unit-tested); the browser wiring is `lib/use-recorder.ts` + `components/recorder.tsx` + `components/level-meter.tsx`; both capture paths share `lib/upload-audio.ts`. The Sessions screen lists takes (newest-first, tabular duration); `/sessions/<id>` plays via `GET …/audio` with HTTP Range (206) and deletes rows + files.
